# 博客文章写作规范（2026-05-24版）

## 一、Front Matter 格式（严格按此格式，一字不差）

```yaml
---
title: '文章标题（引号必须是单引号）'
date: '2026-05-29T08:00:00+08:00'   # T是大写，+08:00必须有，单引号
draft: false
showtoc: true
categories: ['分类名']               # 中括号，单引号
tags: ['标签1', '标签2']              # 中括号，单引号
description: '一句话描述（无引号）'
---
```

**禁止出现的字段：** `author`、`slug`、`典型意义`、`本案索引`

## 二、正文结构（固定四段，不多不少）

```
[开头段落]
故事背景，2-3段，最后一行是一个问句结尾（"这事怎么算？"）
段与段之间空行分开

## 基本案情
叙述型，不要序号，用"某天/随后/最后"串联事件

## 争议焦点
**核心问题：xxx？**
一句话说明核心问题

## 法院认为
分段用加粗小标题引导，如：
**首先，关于xxx。** 内容
**其次，关于xxx。** 内容

## 启示
分段用加粗小标题，如：
**一、xxx。** 内容
**二、xxx。** 内容
**三、xxx。** 内容

---

*本文作者：曹义德律师（华中科技大学法学硕士，前反贪检察官，执业律师）*

*法律咨询：18872778990（微信同号）*
```

## 三、禁止出现的内容

- `**【律师讲故事】**` 开头标签
- 顶部案例名称+法院+案号表格
- `<!--more-->` 标签
- 文档残留字符如 `0120120000000000.000+`
- `## 一句话法治启示` 这样的额外结尾段落
- `*本案索引：`* `*典型意义：*` 等学术摘要
- `## 相关法条：` 单独列出的法条段落
- `## 结语` 单独结尾段落
- `###` 小标题（正文只用 `##` 和加粗）

## 四、categories 分类标准

民事类：['民事纠纷']
劳动类：['劳动争议']
刑事类：['刑事案例']
婚姻家庭：['婚姻家庭']
交通事故：['交通事故']
执行类：['执行案例']
知识产权：['知识产权']

## 五、写作风格要点

- 开头直接讲故事，不解释这是什么案件类型
- 不用"本案是一起xxx纠纷"，直接进入场景
- 启示部分每条都要有具体可操作的建议
- 法条引用融在正文里，不单独列在最后
- 结尾落款两行，顺序固定：曹义德在前，法律咨询在后
